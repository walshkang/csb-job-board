Classify this job listing for a climate-tech job board aimed at MBA candidates.

Title: {job_title_raw}
Company: {company_name}
Location: {location_raw}
Description (truncated): {description_raw}

Climate-tech categories for reference: {category_names}

Return ONLY a single JSON object with these keys:

{
  "job_title_normalized": "standardized title",
  "job_function": "engineering | product | design | operations | sales | marketing | finance | legal | hr | data_science | strategy | policy | supply_chain | customer_success | other",
  "seniority_level": "intern | entry | mid | senior | staff | director | vp | c_suite | unknown",
  "mba_relevance": "low | medium | high",
  "description_summary": "2-3 sentence summary",
  "climate_relevance_confirmed": true/false,
  "climate_relevance_reason": "one sentence if false, else null"
}

## Field rules

job_title_normalized:
- Expand abbreviations: "Sr." -> "Senior", "Eng." -> "Engineer", "Mgr" -> "Manager"
- Standardize level prefixes: "Senior Software Engineer", not "Sr. SWE III"
- Keep the functional title intact — don't invent one

job_function — exactly one of the values shown in the schema above. Jobs must map to exactly ONE function. When multiple could apply, use the tiebreaker ladder below (first match wins):

Tiebreaker priority (highest to lowest):
strategy > product > finance > legal > data_science > policy > marketing > sales > supply_chain > operations > hr > customer_success > design > engineering > other

Deterministic mappings (apply before tiebreaker):
- "Business Development" / "BD" / "Partnerships" / "Sales Ops" / "Account Executive" / "Account Manager" -> sales (always, regardless of pre/post-sale)
- "Customer Success" / "Customer Support" / "Renewals" / "Implementation Specialist" -> customer_success
- "Chief of Staff" / "Strategy & Ops" / "Corporate Strategy" -> strategy
- "Program Manager" / "Project Manager" -> operations (only "Product Manager" or "Technical Product Manager" -> product)
- "General Counsel" / "Legal Counsel" / "Attorney" / "Compliance" -> legal
- "Data Scientist" / "Data Analyst" / "Analytics Engineer" / "ML Engineer" (analytics-focused) -> data_science
- "People Operations" / "Recruiter" / "Talent" / "HRBP" -> hr
- "Corporate Accounting" / "Controller" / "FP&A" / "Treasury" / "Finance Manager" -> finance
- "Policy" / "Government Affairs" / "Regulatory Affairs" -> policy
- "Procurement" / "Logistics" / "Materials" / "Planning" (supply-chain context) -> supply_chain
- "Product Designer" / "UX" / "UI" / "Visual Designer" -> design
- "other" is a last resort — only use if NO listed function fits at all. If partial fit, pick using the tiebreaker.

seniority_level — exactly one of the values shown in the schema above:
- "Manager" with direct reports -> senior
- "Manager" as IC title (e.g., "Account Manager", "Program Manager") -> mid
- "Head of" at < 50-person company -> senior; otherwise director
- "Principal" / "Staff" -> staff
- "Associate" -> entry (unless "Associate Director" -> director)
- "C-suite" (CEO, COO, CFO, CTO, CPO) -> c_suite
- No title clues + 3–5 years experience in description -> mid
- No title clues + 7+ years experience in description -> senior
- "unknown" is the only valid fallback — never omit this field

mba_relevance — exactly one of:
  low, medium, high
- high: strategy, business development, product leadership, GM, operations leadership, venture/finance roles
- medium: product management, marketing leadership, partnerships, supply chain leadership, and adjacent business-oriented roles
- low: primarily technical/IC roles or early-career roles that are less typical for MBA recruiting

climate_relevance_confirmed:
- true if the company operates in one of the provided climate-tech categories OR the job description explicitly references climate, energy transition, sustainability, decarbonization, cleantech
- false if the company/role has no apparent connection to climate tech
- When false, explain why in climate_relevance_reason (e.g., "Company is a generic SaaS platform with no climate focus")

## Examples

Input: Title="VP of Business Development", Company="Arcadia" (clean energy), Location="Remote", 5+ years in energy markets
Output: job_function=sales, seniority_level=vp, mba_relevance=high, climate_relevance_confirmed=true

Input: Title="Junior Frontend Engineer", Company="Climate startup", Location="San Francisco, CA", 1-2 years experience
Output: job_function=engineering, seniority_level=entry, mba_relevance=low, climate_relevance_confirmed=true

Input: Title="Account Manager", Company="SolarEdge", Location="Austin, TX (Hybrid)"
Output: job_function=sales, seniority_level=mid, mba_relevance=medium, climate_relevance_confirmed=true
(Account Manager is always sales per deterministic mapping, regardless of pre/post-sale context)

Input: Title="Customer Support Specialist", Company="Vulcan Elements", Location="Remote"
Output: job_function=customer_success, seniority_level=mid, mba_relevance=low, climate_relevance_confirmed=true

Input: Title="Associate General Counsel, Commercial", Company="GoFlex", Location="New York, NY"
Output: job_function=legal, seniority_level=entry, mba_relevance=low, climate_relevance_confirmed=false

Input: Title="Director, Analytics Engineer", Company="GoFlex", Location="San Francisco, CA"
Output: job_function=data_science, seniority_level=director, mba_relevance=medium, climate_relevance_confirmed=false

Return valid JSON only. No markdown fences.
