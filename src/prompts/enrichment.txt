Classify this job listing for a climate-tech job board aimed at MBA candidates.

Title: {job_title_raw}
Company: {company_name}
Location: {location_raw}
Description (truncated): {description_raw}

Climate-tech categories for reference: {category_names}

Return ONLY a single JSON object with these keys:

{
  "job_title_normalized": "standardized title",
  "job_function": "one of the allowed values",
  "seniority_level": "one of the allowed values",
  "location_type": "one of the allowed values",
  "mba_relevance": "low | medium | high",
  "description_summary": "2-3 sentence summary",
  "climate_relevance_confirmed": true/false,
  "climate_relevance_reason": "one sentence if false, else null"
}

## Field rules

job_title_normalized:
- Expand abbreviations: "Sr." -> "Senior", "Eng." -> "Engineer", "Mgr" -> "Manager"
- Standardize level prefixes: "Senior Software Engineer", not "Sr. SWE III"
- Keep the functional title intact — don't invent one

job_function — exactly one of:
  engineering, product, design, operations, sales, marketing, finance,
  legal, hr, data_science, strategy, policy, supply_chain, other
- "Business Development" -> sales
- "Chief of Staff" / "Strategy & Ops" -> strategy
- "Program Manager" -> operations (unless explicitly product-focused)
- If genuinely ambiguous, use the primary responsibility from the description

seniority_level — exactly one of:
  intern, entry, mid, senior, staff, director, vp, c_suite
- "Manager" with direct reports -> senior
- "Manager" as IC title (e.g., "Account Manager") -> mid
- "Head of" with < 50 employees company -> senior; otherwise director
- "Principal" / "Staff" -> staff
- "Associate" -> entry (unless "Associate Director" -> director)
- No title clues + 3-5 years experience -> mid
- No title clues + 7+ years experience -> senior

location_type — exactly one of:
  remote, hybrid, on_site, unknown
- Infer from BOTH location_raw and description text
- "Remote" or "Anywhere" -> remote
- "Hybrid" or city + "flexible" -> hybrid
- City with no remote mention -> on_site
- No signal at all -> unknown

mba_relevance — exactly one of:
  low, medium, high
- high: strategy, business development, product leadership, GM, operations leadership, venture/finance roles
- medium: product management, marketing leadership, partnerships, supply chain leadership, and adjacent business-oriented roles
- low: primarily technical/IC roles or early-career roles that are less typical for MBA recruiting

climate_relevance_confirmed:
- true if the company operates in one of the provided climate-tech categories OR the job description explicitly references climate, energy transition, sustainability, decarbonization, cleantech
- false if the company/role has no apparent connection to climate tech
- When false, explain why in climate_relevance_reason (e.g., "Company is a generic SaaS platform with no climate focus")

## Examples

Input: Title="VP of Business Development", Company="Arcadia" (clean energy), Location="Remote", 5+ years in energy markets
Output: job_function=sales, seniority_level=vp, mba_relevance=high, location_type=remote, climate_relevance_confirmed=true

Input: Title="Junior Frontend Engineer", Company="Climate startup", Location="San Francisco, CA", 1-2 years experience
Output: job_function=engineering, seniority_level=entry, mba_relevance=low, location_type=on_site, climate_relevance_confirmed=true

Input: Title="Account Manager", Company="SolarEdge", Location="Austin, TX (Hybrid)"
Output: job_function=sales, seniority_level=mid, mba_relevance=medium, location_type=hybrid, climate_relevance_confirmed=true

Return valid JSON only. No markdown fences.
